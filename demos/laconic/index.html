Laconic Grid

A grid system with a little less <class>

Laconic does one thing and does it well: places html elements into columnar sets, i.e. a grid. It goes to up 6 columns and does not have a single full width column because that is stupid and all block elements are full width columns. If you need more than 6 columns, you might want to re-evaluate your design.

Features:

  • Responsive
  • Looks the same in all browsers
  • Semantic
  • Simple to code
  • No janky gutters
  • No vomiting divs all over your html documents (unless you want to do that)
  • Percentage based sizing
  • Negative margins reduce/eliminate browser rounding errors
  • Add your own padding/margins
    • Box-sizing is set to "border-box" so borders, padding and margins don't affect grid alignment.
  • Doesn't break if one section has more content than another.

What does the markup look like?

A 3 column footer would look like the following:

<footer class="clearfix">
    <section class="col-1-3">
        <p>My content here</p>
    </section>
    <section class="col-2-3">
        <p>More of my content</p>
    </section>
    <section class="col-3-3">
        <p>The third column</p>
    </section>
</footer>

The parent section has the class clearfix to clear floats, and the sections themselves use the column classes .col-1-3, .col-2-3, .col-3-3 You can place column classes on nearly any element, such as unordered lists, divs, and images.

Colorful boxes and code samples don't give the whole picture though. Try opening this site in different browsers and platforms, or check it out on screenfly.
